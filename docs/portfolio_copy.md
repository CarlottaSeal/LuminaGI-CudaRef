## Description

LuminaGI-CudaRef is a CUDA path tracer built as the offline reference for LuminaGI, my real-time D3D12 GI engine. Pressing F9 in LuminaGI dumps the current frame's scene and screenshot to disk; the CUDA renderer takes that scene, path-traces a reference image, and runs an image diff (PSNR / SSIM / heatmap) against the engine output.

The point isn't the renderer itself — it's having a quantitative answer to "is this frame correct?". The diff already caught two silent bugs in the engine's export path before any optimization work began.

## Main Technical Points

### Reference Pipeline

`F9 in LuminaGI → scene.json + screenshot.png → cuda_ref.exe → ref.png → diff.py → HTML report`

Engine side is a small instrumentation hook: `Scene::DumpToJSON` writes triangle soup + lights + camera in world space, the F9 handler defers the capture to after EndFrame (calling it mid-frame resets the command list and crashes the next BindConstantBuffer), and `AutomatedTesting --screenshot` auto-dumps a matching `.json`. Everything downstream is independent and runs on any frame, current or historical.

### Host-side scene loader and BVH

JSON parser (nlohmann::json) → triangle soup with materials → LBVH built CPU-side using Morton codes and radix split, then BFS-relayed for cache-friendly traversal order. 1.05 M triangles → 810k BVH nodes, 150 ms build. Test scene: 62 meshes, 1 sun + 2 point lights, 1728×864 output.

### CUDA Path Tracer

Camera rays, BVH traversal, Möller–Trumbore intersection, shadow rays, cosine-weighted hemisphere sampling for indirect, Russian roulette termination, progressive accumulation. Diffuse texture sampling via CUDA texture objects (sRGB decode + bilinear). Render times on RTX 4080 Laptop: 4 ms direct only, 3.0 s at 64 spp / 2 bounces, 8.7 s at 256 spp / 2 bounces.

Final PSNR vs engine: 21.8 dB / SSIM 0.631 at 64 spp / 2 bounces. The remaining gap is systematic — the engine has an ambient term and normal-map detail this reference doesn't model. The tool is designed to detect *changes* in error, not absolute fidelity.

### Profiling and Optimization

Profiled the main kernel with Nsight Compute — L2-bandwidth bound, 90.8% L2 throughput, 7.2% DRAM. Added `__launch_bounds__(256, 4)` to drop register pressure (76 → 64 per thread), lifting achieved occupancy from 46.6% to 61.8% for ~5–10% real speedup. Two ncu-suggested leads (shared-memory BVH cache, ray-sort between bounces via `thrust::sort_by_key`) were implemented and benchmarked, but both regressed on cached workloads — kept toggleable, off by default.

### Neural Denoiser (UNet on 8-spp output)

256 spp takes ~13 s/frame, most of which is beating down Monte Carlo noise. Trained a small UNet (4-level, 32 base channels, 1.9 M params, residual on input, L1 loss) on (8-spp noisy, 256-spp clean) pairs from 50 random camera poses in this scene.

Held-out camera, vs 1024-spp ground truth:

- 8 spp raw: 400 ms, PSNR 34.52 dB, SSIM 0.815
- 8 spp + UNet: 597 ms, PSNR 42.04 dB, SSIM 0.964
- 256 spp baseline: 13,400 ms, PSNR 49.57 dB, SSIM 0.991

About 22× faster than the 256-spp baseline for ~80% of its PSNR gain over raw 8 spp. SSIM 0.964 is visually indistinguishable at typical viewing distance. Trained scene-specific (single indoor room) — deliberately not a general-purpose denoiser. Dataset generated by `cuda_ref --batch` (renders both passes per scene-load); training and inference in PyTorch.

### Tech

CUDA 13, Nsight Compute, Nsight Systems, cuobjdump, Thrust, LBVH (Morton + radix split), Möller–Trumbore, cosine-weighted hemisphere, Russian roulette, PyTorch UNet, PSNR / SSIM diff. Hardware: RTX 4080 Laptop, Ada SM 8.9.
